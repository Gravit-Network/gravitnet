# Contributing to Gravit Core Canon

Thank you for considering contributing to the Gravit Trust Protocol specification.

This repository maintains the **canonical specification** — a formal, mathematical foundation for computable trust infrastructure. Contributions help evolve the protocol in a transparent, rigorous way.

## Ways to Contribute

- Propose changes or additions via **Gravit Canon Proposals** (GCP) — the only mechanism for modifying the canon
- Submit research, simulations, philosophical essays or threat analyses
- Improve documentation, fix typos or clarify mathematical expressions
- Report inconsistencies, ambiguities or potential vulnerabilities in the spec
- Participate in reviews of open GCPs (provide feedback, simulations, counter-arguments)

## How to Get Started

1. Read the main README.md and [/gcp/README.md](/gcp/README.md)
2. Familiarize yourself with the current canon: /math-core, /ontology, /epistemology etc.
3. Check open issues and active GCPs for discussion
4. For normative changes → follow the GCP process (use template)
5. For discussion / questions → open an issue with appropriate template
6. Submit Pull Requests for small fixes or markdown improvements

## Proposal-First Principle

**All changes to canonical documents (/math-core, /ontology, etc.) must go through a GCP.**
Direct edits to core files without a proposal will not be merged.

See: [/gcp/README.md](/gcp/README.md) and [/governance/gcp-process.md](/governance/gcp-process.md)

## Communication

- Use GitHub Issues for technical discussion
- Use Pull Requests for concrete changes
- Respect the [Code of Conduct](CODE_OF_CONDUCT.md)

## Recognition

Significant contributions (merged GCPs, high-quality reviews, simulations) may be recognized in author lists, acknowledgements or future reputation bootstrapping (self-application of the Gravit trust model to contributors).

Thank you for helping build transparent trust infrastructure.
